\hypertarget{md_README_autotoc_md1}{}\doxysection{Descripción}\label{md_README_autotoc_md1}
Proyecto hecho bajo el marco de la materia {\itshape Sistemas en Tiempo Real}, que consiste en el manejo a través de una interfaz gráfica de la velocidad de un motor. El proyecto consiste en tres módulos\+:


\begin{DoxyEnumerate}
\item Interfaz\+: implementada en python3, pone a disposición del usuario la modificación de la velocidad deseada, y las variables P, I, D del módulo P\+ID, que se encarga de calibrar el torque del motor.
\item Tiempo Real\+: que combina un módulo de P\+ID para calibrar el torque del motor en función de la velocidad actual y la deseada, un módulo R\+TC para realizar controles cíclicos del motor, y un módulo de control del motor. Éste módulo de control del motor está implementado de dos formas\+:
\begin{DoxyItemize}
\item Un simulador del motor.
\item Un controlador del motor que se comunica por U\+DP con el motor real.
\end{DoxyItemize}
\item El motor real, que se utiliza sólo si se usa el módulo de {\itshape tiempo real} correspondiente, y que se espera está expuesto a través de un servidor a Internet, esperando comandos por U\+DP.
\end{DoxyEnumerate}\hypertarget{md_README_autotoc_md2}{}\doxysection{Protocolo de comunicación}\label{md_README_autotoc_md2}
La interfaz envía por el socket T\+CP una cadena de caracteres con el formato\+: \textquotesingle{}velocidad deseada,kp,ki,kd\textquotesingle{} Donde la velocidad deseada es un entero en el rango inclusivo \mbox{[}0,255\mbox{]}, y kp, ki, kd son números de punto flotante.

El servidor envía a la interfaz como respuesta a dicha información la cadena de formato\+: \textquotesingle{}velocidad real,torque real\textquotesingle{} Donde ambos son enteros en el rango inclusivo \mbox{[}0,255\mbox{]}.

El servidor envía al motor la cadena con el formato\+: \textquotesingle{}torque,{\itshape valor}\textquotesingle{}
